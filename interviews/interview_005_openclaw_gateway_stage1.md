## Контекст (30 секунд)

Сейчас ядро отвечает только на явные локальные ходы `openclaw agent --local` с этой машины.
**Гейтвей** — постоянный локальный демон OpenClaw — добавляет: Control UI (чат с ядром из браузера с
любого устройства tailnet), cron/heartbeat (ядро само просыпается по расписанию), webhooks, каналы
(Telegram и т.п.), подключение Android-ноды (Q2 инт. 004 — наш будущий клиент). Публичного интернета
это НЕ требует и НЕ открывает: доступ строго внутри Tailscale-сети (правило «голос/ядро только
tailnet», инт. 003 Q5). Латентность и модели не меняются — это тот же llama-swap.

### Q2. Подтверждения exec — эскалировать со ступени 0?

Сейчас: `security: allowlist, ask: off` — незнакомая команда молча отклоняется. С гейтвеем появляется
канал, куда ядро может присылать запрос «разрешить команду X?» (approve/deny кнопкой).

- **A) (рекомендую)** **Оставить allowlist/deny как есть**, пересмотреть после обкатки гейтвея.
  *Почему: минимальная поверхность, ноль внезапных диалогов; эскалация — отдельным решением, когда
  появится реальная потребность.*
- **B)** Включить **ask on-miss** сразу с гейтвеем: незнакомая команда → запрос подтверждения в
  Control UI. *Гибче (ядро сможет больше по одному клику), но каждый промах дёргает владельца.*
