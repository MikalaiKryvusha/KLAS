# /pause — завершение рабочей сессии

Человек ставит паузу. Выполни рутину завершения **по порядку**. Кратко рассказывай каждый шаг в чате.
Не пропускай шаги. Если шаг падает — остановись, скажи человеку, не продолжай вслепую.

## Шаг 1. Запиши статус и планы в STATUS.md

Обнови `STATUS.md`:
- **Что сделано за сессию** — конкретно, с привязкой к багам/фичам и файлам.
- **Текущая позиция** — что работает, что в процессе, где мы.
- **Планы на следующую сессию** — ясный чек-лист «где продолжить» (команды, файлы, что проверить
  первым). Помни: следующая сессия агента стартует с пустым контекстом — пиши так, чтобы она сразу
  включилась.
- Переводи относительные даты в абсолютные (текущая дата — из контекста / `date`).

Сверься с активными документами багов в `bugs/` и отрази их статус.

## Шаг 2. Освежи README

Приведи `README.md` (если он есть) в соответствие с реальностью: статусы фаз, работающие возможности,
инструкции. Не выдумывай — отражай только реально сделанное и проверенное.

## Шаг 3. Проверь состояние стека

Классической сборки в проекте нет; вместо неё — проверка, что ничего не оставлено сломанным:
если в сессии трогался LLM-стек — `curl http://127.0.0.1:8080/health` (жив/осознанно остановлен),
изменённые скрипты/конфиги прокомментированы, изменения отражены в картах. Если что-то сломано —
остановись и покажи, не коммить заведомо сломанное состояние без пометки.

## Шаг 4. Commit и push

```
git add -A && git commit -m "..."
```

Стиль сообщения (из `AGENT_GUIDE.md`): `feat:` / `fix:` / `docs:` / `refactor:` / `ci:` + одна строка.
Заверши сообщение:
```
Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>
```

Push — только если настроен remote (пока его нет — см. interview_001; просто скажи об этом в отчёте).

## Шаг 5. Отчитайся

Кратко отчитайся человеку: что записано, что проверено, хеш коммита, что запушено (или почему нет), и
одна строка — главный план следующей сессии. Это и есть пауза.

## Заметки

- Если push отклонён (non-fast-forward) — `git pull --rebase`, повтори push, затем сообщи человеку о
  расхождении.
- Тяжёлые артефакты (модели GGUF и т.п.) в git не входят — это нормально (`.gitignore`).
