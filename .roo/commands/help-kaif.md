# /help-kaif — объяснить оператору KAIF, в чате

Выдай **пользовательский мануал по KAIF прямо в чат**, на рабочем языке оператора (русский). Это
обучающий момент для человека, ведущего проект, — скилл **не меняет файлы**, только даёт ясное,
хорошо структурированное объяснение, по которому можно действовать.

## Рамка (важно)
- KAIF **уже развёрнут** в этом проекте. **Не** рассказывай про распаковку/установку — это сделано.
  Говори в духе «вот как *пользоваться* тем, что уже здесь».
- «Что такое KAIF» — **пара предложений**. Основной объём ответа — **как пользоваться**: структура,
  конвенции, документы, директории и скиллы.
- Пиши на рабочем языке оператора. Имена `/команд` и файлов — канонические.
- Опирайся на развёрнутую реальность *этого* проекта (читай `KAIF_FRAMEWORK.md`, `AGENT_GUIDE.md`,
  README директорий) — не на общий питч. Терминологию адаптируй к сфере проекта
  (ИИ-инфраструктура: стек, движок, модель, профиль, харнесс).

## Что выдать (структурируй сообщение так)

1. **Что такое KAIF (2–3 предложения).** Контекстоустойчивый, дисциплинированно-автономный метод для
   тандема человек–ИИ: человек — визионер, агент — исполнитель, а память/дисциплина проекта живут в
   файлах репозитория, чтобы ни одна сессия не начиналась с нуля. Одна строка — почему это полезно
   здесь.

2. **Ключевые документы — что читать/вести, и чей каждый.** Кратко, списком:
   `AGENT_GUIDE.md` (канон), `PHILOSOPHY.md` (как мыслит агент), `BUG_FIXING_FRAMEWORK.md`,
   **`GOAL.md`** (видение владельца — *твой* документ), `STATUS.md` (живое состояние),
   `MASTER_PLAN.md` (дорожная карта), внешняя и внутренняя карты, `KAIF_FRAMEWORK.md` (резюме «что
   развёрнуто»).

3. **Директории — где живёт знание и где действует владелец.** `plans/`, `ideas/` (в основном твоя),
   `bugs/`, `researches/`, `interviews/` (отвечаешь здесь), `homeworks/` (задачи тебе). Одной строкой —
   конвенция DONE-тега.

4. **Скиллы — команды, которые ты набираешь.** Перечисли группами, каждый с одной строкой назначения:
   сессия (`/resume`, `/pause`), автономия (`/autoloop`, `/dayloop`, `/nightloop`), гигиена
   (`/refresh-context`, `/check-backlog`), знание (`/report-bug`, `/bug-research`, `/propose-idea`),
   владелец (`/interview`), планирование (`/revision`), помощь (`/help-kaif`), релизы (`/release`) и
   жизненный цикл (`/kaif-version`, `/kaif-update`, `/kaif-fork`, `/kaif-switch-origin`,
   `/kaif-remove`).

5. **Как выглядит нормальный рабочий процесс.** Короткий пример: *«`/resume` для старта → я работаю и
   держу `STATUS.md` актуальным → ты кладёшь идеи в `ideas/` или отвечаешь на `/interview` → `/pause`
   для завершения.»* Отметь роль человека (визионер: `GOAL.md`, идеи, ответы на интервью) и агента
   (исполнитель).

6. **Куда идти глубже.** Укажи на `KAIF_FRAMEWORK.md` и `AGENT_GUIDE.md` за полной детализацией.

## Заметки
- Это скилл **прочитай-и-объясни** — не редактируй файлы, не разворачивай, не меняй состояние.
- Держи сканируемость: короткие секции, списки вместо абзацев. Цель — оператор дочитал и точно знает,
  какой документ открыть и какую команду набрать следующей.
- Если оператор спросил про одну конкретную часть («как работают интервью?») — ответь сфокусированно,
  затем предложи полный мануал.
