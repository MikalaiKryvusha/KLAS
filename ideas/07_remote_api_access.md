## 1. Удалённый доступ по API (главная фича)

В KLAS обязательно должен быть **удалённый доступ по API** через интернет: чтобы с любого компьютера
подключить KLAS как агентскую систему в **Zoo Code** (форк Cline, «Зоопарк»). Запросы ходят на
домашний ПК владельца, а сам он вайб-кодит откуда угодно удалённо.

Желательно — по обычному **URL**, чтобы не обязательно было логиниться в Tailscale: достаточно знать
адрес и пароль API.

## 7. Анонимное разворачивание KLAS

По аналогии с KAIF 1.2 (там появился метод анонимного разворачивания фреймворка) — реализовать
скрипт **анонимного разворачивания KLAS** на любом компьютере. Любые упоминания имени автора
(Николай Кривуша / MikalaiKryvusha) и ссылок на его GitHub должны быть удалены из всех файлов
локально развёрнутого KLAS, чтобы личность автора нельзя было установить. Все git-ссылки на origin
тоже выжигаем — при анонимном разворачивании проект больше не обновляется из origin.

В отличие от KAIF, KLAS не упакован в один документ — значит, у нас гораздо больше свободы: можно
писать какие угодно скрипты разворачивания, т.к. KLAS копируется классически через `git clone`.

## 8. Итог: релиз MVP KLAS 0.1 — First KLAS

Все задания из этого документа берём в работу, реализуем и по итогу оформляем **релиз MVP KLAS 0.1
(First KLAS)** — минимально рабочая версия продукта, которую можно скачать и развернуть (обычно или
анонимно). Она сама:
- проверяет зависимости,
- докачивает нужные тяжёлые библиотеки,
- настраивает docker, порты — всё нужное, автоматически, консольно.

**Что НЕ упаковываем** в релиз/репозиторий (докачивается локально скриптами при разворачивании):
- тяжёлые вещи вроде моделей;
- сторонние open-source модули (ссылаемся и умно докачиваем из их origin в интернете);
- `.zim`-библиотеки википедии (безумно тяжёлые) — для них вообще отдельный **опциональный** скрипт,
  запускаемый пользователем по желанию.
