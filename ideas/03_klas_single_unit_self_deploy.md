## Суть (слова владельца, структурировано агентом)

1. **Полностью избавиться от `F:\LOCAL_NOMAD\`** — ✅ сделано: каталог удалён владельцем (2026-07-04),
   всё нужное живёт в KLAS.
2. KLAS оформляется как **одна законченная работающая единица**, разворачиваемая на новый пустой ПК
   **двумя командами**:
   - `git clone https://github.com/MikalaiKryvusha/KLAS.git`
   - «развернуть» (одна команда, например `node tools/deploy.mjs`)
3. **Репозиторий — обёртка / каркас / фрейм**: документы, конфиги, скрипты, скиллы. Тяжёлое
   (движок, модели, docker-образы, база знаний) в git НЕ пакуется.
4. При «развернуть» система **сама скачивает и разворачивает** всё тяжёлое из сети — без ручных шагов.

## Что это означает практически (набросок агента — уточнить в плане)

| Компонент | В git (каркас) | Скачивает deploy |
|-----------|----------------|------------------|
| Документы KAIF, скиллы, tools | ✅ | — |
| Профили запуска (bat), docker-compose | ✅ | — |
| llama.cpp (CUDA-сборка) | манифест версии | ✅ релиз с GitHub |
| GGUF-модели | манифест (имя, quant, URL, SHA) | ✅ с Hugging Face |
| docker-образы (kiwix, homepage, caddy) | compose-файл | ✅ `docker compose pull` |
| .zim википедии (kiwixdb) | манифест | ✅ с kiwix.org |

- `tools/deploy.mjs` — идемпотентный (повторный запуск дотягивает недостающее), с dry-run,
  проверкой хешей и предпроверками (node, git, docker, NVIDIA-драйвер).
- Текущая машина владельца — первый полигон: deploy должен «дотянуть» уже развёрнутую систему
  без разрушения (те же манифесты, ничего не перекачивает зря).
