# PAI — Психо-Акустический Интерфейс (кластер документов)

Подидея/модуль [Jarvis (idea 05)](../05_Jarvis.md): режим, где Jarvis по голосовому запросу формирует и
проигрывает в наушники **психоакустические протоколы** — аудиопотоки, влияющие на состояние (сон, фокус,
расслабление) или несущие информацию/уведомления (акустический CLI). Уши как `adb`-порт к мозгу.

Статус: 💡 **НА БУДУЩЕЕ** (после базы KLAS и базового Jarvis). Одобрена по факту авторства владельца.

## Ключевые выводы (одним взглядом)

- PAI — **ассистивный аудио-UX** («звуковой ландшафт настроения» + удобные звуковые уведомления), **не**
  «программирование мозга» и **не** медицина. Доказательность entrainment — смешанная/умеренная; тон
  коммуникации честный.
- **Безопасность — слой Фазы 1, не опция:** лимит громкости, дисклеймер, осторожность с эпилепсией и
  частотами <10 Гц, контекст-чек (не за рулём/не на ходу).
- **Стек:** свой генератор на **Web Audio/Tone.js** (просто, MIT-совместимо) — приоритет; SBaGenX/Gnaural
  (GPL) — референс/внешний движок; SuperCollider — только под тяжёлую сонификацию.
- **Ультразвук (LIFU/tFUS) — вне скоупа** (специальное оборудование, медконтекст).
