# Idea 06 — try-model: простой механизм «скачал модель → проверил бенчем» + регулярный поиск новинок

> Источник: **владелец** (интервью 002, 2026-07-03): «нужно периодически гуглить свежие треды,
> энтузиасты часто запускают новые крутые модели… нужно иметь в KLAS простой механизм скачивания
> и проверки модели бенчмарком». Статус: ✅ ОДОБРЕНА по факту авторства — реализация v1 сразу.

## Суть

1. **Одна команда — попробовать любую модель с Hugging Face:**
   `node tools/try-model.mjs <hf-repo> <файл.gguf>` — скачивает (с докачкой) в
   `LLMs/LLAMACPP_MODELS/`, гонит скоростной бенч (`bench-model.ps1`) и печатает готовый сниппет
   для llama-swap (добавил → `node tools/agent-bench.mjs <alias>` меряет «ум»).
2. **Периодический поиск новинок:** пункт автономного беклога — раз в неделю-две агент ищет в вебе
   свежие сильные модели ≤16 GB VRAM (r/LocalLLaMA, HF trending, обзоры), достойных кандидатов
   прогоняет этим пайплайном и вносит в таблицу `researches/02`.
