# `ideas/` — предложения фич и улучшений

Детальные идеи *что* строить — обычно узкий срез проекта, описанный достаточно хорошо, чтобы агент мог
реализовать по нему. Чаще всего пишет **человек**, но агент тоже предлагает идеи. Один `NN_<name>.md`
на идею.

**Человеку (владельцу):** это твоя главная авторская директория. Клади сюда идею с описанием, чего
хочешь; агент приведёт её к чистой структурированной форме и реализует по ней. Идея — часть
**видения** продукта; агент реализует её только после твоего одобрения.

**ИИ-агенту:** читай идеи владельца, исправляй опечатки, минимально реструктурируй для ясности, затем
реализуй. Когда стоящая идея появляется у *тебя* — оформи её здесь со статусом «❓ ждёт одобрения
владельца» (скилл: `/propose-idea`) и **не** реализуй до одобрения. После реализации идеи впиши статус
и дату обратно в её файл и поставь тег `DONE` (`git mv NN_x.md NN_DONE_x.md`).
