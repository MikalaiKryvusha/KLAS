# Idea 01 — Рождение KLAS: ребрендинг и миграция системы в F:\KLAS

> Источник: **владелец** (Николай Кривуша), 2026-07-02.
> Статус: 🔧 В РАБОТЕ — идея владельца = элемент видения, одобрена по факту авторства.
> Оформлено агентом 2026-07-02: текст владельца сохранён, поправлены опечатки, добавлена структура.

## Суть

Проект локальной self-hosted ИИ-системы получает собственное имя — **KLAS (Krinik Local Agent
System)** — и переезжает из `F:\LOCAL_NOMAD\` в `F:\KLAS\`.

Владелец делает проект локальной self-hosted ИИ-системы: ИИ-агент на базе LLM на геймерской GPU +
веб-дашборды + база знаний в локальной википедии. Вдохновлялся проектом Local Nomad и изначально так
и назвал директорию на компьютере — `LOCAL_NOMAD`. Но понял, что хочет **свою** систему — из этого
родилось название KLAS.

Описание системы (авторское, каноническое):

> KLAS (Krinik Local Agent System) is a self-hosted AI ecosystem powered by a local LLM on gaming GPU.
> It features autonomous AI agents, a web dashboard for control, and a localized Wikipedia-style
> knowledge base for total data privacy and offline efficiency. KAIF-powered framework for homelab
> enthusiasts.

## Принципы (от владельца)

- **Не стесняемся заимствовать:** активно используем открытые GitHub-репозитории с инструментами и
  фреймворками, которые уже решают части наших задач по KLAS.
- **Технологии:** JS, современный **.mjs + Node** — для скриптов, инструментов, бекенда и БД.

## План действий (от владельца)

- ✅ Репозиторий создан: https://github.com/MikalaiKryvusha/KLAS
- ✅ Директория на ПК под систему создана: `F:\KLAS\`
- **Прежде чем начать** — фреймворк KAIF, который уже развёрнут под изначальный `GOAL.md`,
  адаптировать под новую идею — KLAS. Сам `GOAL.md` переписать как **симбиоз** его текущего
  содержимого и этого документа-идеи.
- Написать скрипт, который **УМНО перемещает** всё необходимое из `F:\LOCAL_NOMAD\` в `F:\KLAS\`:
  - берём только действительно нужное; что устарело, не нужно или будет сделано иначе другими
    инструментами — с собой не берём;
  - скрипт также переименовывает документы самого Claude Code по работе над текущим проектом,
    меняет все пути, все имена, память по чатам — полностью мигрирует уже начавшего работу
    Claude Code с `F:\LOCAL_NOMAD\local_ai_agent\` на новое место (переносится «личность»
    Claude AI Fable 5).
- **Запускает скрипт владелец сам.** Ожидание после миграции: при следующем запуске VS Code (уже в
  `F:\KLAS\`) агент не теряет память о начатой работе, внутренние документы корректно мигрировали,
  пути и имена корректно заменились.

## Как вписывается в мастер-план и видение

Это уточнение самого видения (`GOAL.md` переписывается по указанию владельца) + новая веха в
`MASTER_PLAN.md` (миграция в KLAS). Функционально расширяет цель: база знаний (локальная википедия —
kiwix уже есть в окружении) и веб-дашборд становятся явными частями продукта KLAS.

## Реализация

1. ✅ Идея оформлена (этот документ).
2. `GOAL.md` — симбиоз (сделано, см. файл).
3. KAIF адаптирован под KLAS: `AGENT_GUIDE.md` (идентичность), `MASTER_PLAN.md` (/revision),
   `STATUS.md`, карты, `KAIF_FRAMEWORK.md`, `CLAUDE.md`/`AGENTS.md`, `package.json`.
4. План миграции с манифестом «берём/оставляем»: `plans/01_klas_migration.md`.
5. Скрипт миграции: `tools/migrate-to-klas.mjs` (dry-run по умолчанию, `--apply` для выполнения).
6. Владелец запускает скрипт → следующая сессия в `F:\KLAS\` проверяет целостность миграции,
   настраивает remote `origin` → push → идея DONE.

## Открытые вопросы / что требует ответа человека

  умолчанию записаны в манифесте плана; владелец может поправить манифест до запуска.
