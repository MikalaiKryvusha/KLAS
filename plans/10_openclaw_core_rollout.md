## Ступень 1 — гейтвей в tailnet ✅ (решения — интервью 005, все A; выполнена 2026-07-17)

Как развёрнуто: `gateway` в `~/.openclaw/openclaw.json` — `mode: local, bind: tailnet (100.80.125.66),
port: 18789, auth: token` (токен — `caddy/PASSWORD.local.txt`, вне git); Control UI —
`http://100.80.125.66:18789/` (с устройств tailnet; вход по токену). Запуск/останов встроен в
`tools/klas.ps1` up/down (тихий node-процесс `gateway run`, идемпотентно). Постоянный сервис —
опция владельца: `openclaw gateway install` (Scheduled Task; агент persistence не создаёт).
Проверено: listener на tailscale-IP ✅, Control UI отдаётся и требует токен ✅ (web-shot), агентный
ход ок ✅, в системном промпте только shortlist-навыки ✅.

- **Гейтвей поднимается сейчас** (Q1=A): демон ТОЛЬКО на loopback; доступ с устройств — только
  внутри Tailscale-сети (никакого 0.0.0.0/funnel). Control UI — «пульт ядра» в браузере.
- **Exec НЕ эскалируется** (Q2=A): остаётся `security: allowlist, ask: off`; пересмотр — отдельным
  решением после обкатки гейтвея.
- **Навыки — минимальный shortlist** (Q3=A): включены только нужные сейчас — `healthcheck`
  (самопроверка окружения), `skill-creator` (CRUD-цикл навыков — суть идеи 05); остальные
  выключены. Каждый новый навык включается осознанно и ДОПИСЫВАЕТСЯ в этот список.
- **Канал — только Control UI** (Q4=A): мессенджеры не подключаем.
- Операционка: гейтвей встроен в жизненный цикл KLAS (`tools/klas.ps1` up/down поднимает/гасит его
  вместе со стеком).
