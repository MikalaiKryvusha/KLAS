## Ступень 0 — рабочий профиль, только embedded ✅ (выполнена этим планом, 2026-07-17)

Что включено:
- **Дефолтный профиль** `~/.openclaw/openclaw.json` (без `--profile`): провайдер `klas`
  (llama-swap `http://127.0.0.1:8080/v1`), модели qwen3.5-35b (+ `qwen3.5-35b-r` — ретрай-алиас
  того же инстанса) и gemma-4-12b; `compat.unsupportedToolSchemaKeywords:["pattern"]` обязателен;
  `fallbacks: [qwen3.5-35b-r, gemma-4-12b]`. Шаблон конфига — `openclaw/openclaw.json.example` (git).
- **Exec в жёсткой рамке:** `tools.exec.mode: "allowlist"`, `ask: "off"` — только безопасные
  бинари OpenClaw (safeBins), промах = отказ без вопросов (спрашивать некому: гейтвея/каналов нет).
- **Дисциплина тул-коллов** в `~/.openclaw/workspace/TOOLS.md` (обход peg-бага llama.cpp #20260).
- **Гейтвей, каналы, cron, webhooks — ВЫКЛЮЧЕНЫ** (не запускаем демон вообще). Использование —
  только явные ходы `openclaw agent --local ...` с этой машины.
- **Пилот-профиль остаётся песочницей бенчей** (`--profile pilot`, отдельный воркспейс) — фикстуры
  бенча не мусорят рабочую память ассистента. `tools/openclaw-chain-bench.mjs` гоняет pilot.

Проверка ступени (смоук): обычный ход ✅, цепочка read→write с токеном ✅, exec несейф-команды
отклонён ✅ (см. STATUS 2026-07-17).
