# План 14 — Голосовой тракт Г4: «Голос в кармане» (Android-нода OpenClaw в tailnet)

> Фаза Г4 голосового блока. Зонтик — [plans/07](07_jarvis_gateway_and_client.md); мозг — ядро
> OpenClaw за tailnet-гейтвеем ([plans/10](10_openclaw_core_rollout.md) ступень 1). Решения:
> интервью 004 Q2 (клиент = Android-нода OpenClaw), интервью 005 Q4 + **видение владельца:
> основной канал — Android-нода с ГОЛОСОВЫМ управлением; аутентификация — голос и паттерны
> владельца** (связь: [plans/09 identity/trust](09_jarvis_identity_trust.md), идея 14).
> Статус: 🔲 не начат (после Г3; это ступень 2 выката ядра — перед стартом сверка с владельцем).

## Цель

Голосовой доступ к KLAS **с телефона из любой точки**: Android-нода OpenClaw в tailnet подключена
к гейтвею, Voice Wake/Talk Mode работают; речевой тракт — наш локальный (Г1/Г2) настолько, насколько
позволяет архитектура ноды; всё строго внутри tailnet.

## Шаги

- [ ] **Исследование ноды (первый шаг, до установки):** как Android-нода OpenClaw работает с
      аудио — где STT (на устройстве/на хосте?), каким TTS озвучивает (док смоука: ElevenLabs/
      системный ⚠️), можно ли завести НАШ тракт (Silero/GigaAM на хосте) в её цикл; что с
      wake word. Результат — дополнение в research 09 §3.4 + уточнение шагов ниже.
- [ ] **Телефон владельца в tailnet** (Tailscale-клиент уже в экосистеме KLAS) + установка ноды
      (🏠 совместно с владельцем — его устройство).
- [ ] **Подключение к гейтвею** (ws://100.80.125.66:18789, токен из PASSWORD.local.txt), пейринг
      ноды по правилам OpenClaw (approve устройства).
- [ ] **Talk Mode / Voice Wake** — проверить голосовой ход до ядра и обратно; замерить латентность
      на сотовой сети (метрика для решения о WebRTC/Pipecat в Г5).
- [ ] **Русский голос ответа на телефоне:** если нода озвучивает только облаком/системой — задать
      системный русский TTS Android как временный; интеграция нашего Silero-тракта — по итогам
      исследования (возможно через кастомный TTS-эндпоинт ноды или ответ аудиофайлом).
- [ ] **Безопасность:** нода видна только в tailnet; exec-полномочия ядра НЕ расширяются
      (allowlist ступени 0 действует для всех каналов); аудит скоупов ноды (камера/экран/команды
      устройства — ВЫКЛ до отдельного решения владельца).
- [ ] **Задел Г5/идентичности:** зафиксировать точки, где в канал встанет голосовая аутентификация
      (plans/09): пока нода аутентифицируется токеном+tailnet, голос-биометрия — следующий слой.

## Проверка (критерий готовности)

С телефона (сотовая сеть, вне дома, tailnet включён): голосом задать вопрос — получить голосовой
ответ ядра по-русски; чужое устройство без tailnet+токена доступа не имеет; латентность записана.

## Риски

- TTS/STT ноды могут быть жёстко завязаны на облачные/системные сервисы — тогда наш локальный
  тракт входит позже через Г5 (Pipecat/WebRTC своя сессия), а нода остаётся текст+системный голос.
- Ступень 2 полномочий: любые новые скоупы устройства — только через `/interview`.
