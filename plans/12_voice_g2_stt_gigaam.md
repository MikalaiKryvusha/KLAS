# План 12 — Голосовой тракт Г2: «Уши» — русский STT (sherpa-onnx + GigaAM-v3)

> Фаза Г2 голосового блока. Зонтик — [plans/07](07_jarvis_gateway_and_client.md), предыдущая —
> [Г1 «Голос»](11_voice_g1_tts_silero.md). Решения: интервью 004 (sherpa-onnx + GigaAM-v3 вместо
> Whisper.cpp — SOTA по русскому, MIT, стриминг, нативно Windows).
> Статус: 🔲 не начат (после Г1).

## Цель

KLAS **понимает русскую речь локально**: `node tools/voice-hear.mjs <файл.wav>` возвращает текст
(sherpa-onnx + GigaAM-v3, CPU/ONNX, офлайн); точность и скорость замерены на автономном round-trip.

## Шаги

- [ ] **sherpa-onnx под Node:** пакет `sherpa-onnx-node` (официальные prebuilt под win-x64) в
      `tools/` (или сайдкар-CLI sherpa-onnx, если npm-путь окажется хрупким — решить по факту).
- [ ] **Модель GigaAM-v3** (ONNX, ~сотни МБ) → `F:\KLAS\voice\models\`; вариант CTC для старта
      (быстрее), RNNT — если качество/стриминг потребуют.
- [ ] **Харнесс** `tools/voice-hear.mjs <wav>`: текст + тайминги (загрузка/распознавание/RTF).
- [ ] **Round-trip тест** `tools/voice-roundtrip.mjs "<фраза>"`: Г1-синтез → Г2-распознавание →
      сравнение с исходной фразой (нормализация регистра/пунктуации). Автономный критерий качества
      всего аудио-контура без микрофона и человека. Строка в харнесс-таблицу `AGENT_GUIDE.md`.
- [ ] **Стриминг с микрофона** (задел Г3): `tools/voice-hear.mjs --mic` — частичные результаты в
      консоль; проверка работы sherpa-onnx streaming API на Windows.

## Проверка (критерий готовности)

Round-trip ≥ 90% совпадения слов на 10 разнообразных русских фразах (числа, имена, техтермины);
распознавание файла быстрее реального времени на CPU; `--mic` выдаёт частичные результаты.

## Риски

- Prebuilt sherpa-onnx-node под Windows может отставать от версии ядра — тогда сайдкар-exe
  (официальные релизы sherpa-onnx) + вызов из Node.
- GigaAM-v3 лицензия весов — MIT (проверено в research 09), но перепроверить файл лицензии при
  скачивании (правило живости/чистоты OSS).
