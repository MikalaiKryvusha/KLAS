# План 05 — Умный мастер-установщик KLAS «под ключ» (идея 12)

> Реализует `ideas/12_friendly_interactive_installer.md` (+ вплетает идею 13 — ярлыки). Статус:
> 🔬 ПРОЕКТИРОВАНИЕ → РЕАЛИЗАЦИЯ ПО ФАЗАМ (2026-07-04). Наращивается ПОВЕРХ готового движка
> `tools/deploy.mjs` (манифест, идемпотентность, докачка, предпроверки — план 02): его не
> переписываем, а оборачиваем интерактивным мастером и вызываем его шаги.

## Видение (слова владельца)

Супер-удобный, супер-уважительный, с защитой от дурака, предусмотрительный, устойчивый,
**мультиязычный установщик-мастер**. Проводит самого неопытного пользователя за руку от `git clone`
до живого KLAS: выбор языка → детект железа (нужна ли CUDA/ROCm) → интерактивные вопросы
(анонимность, библиотеки, что качать) → установка с внятными пояснениями → **переживает
перезагрузки** и продолжает с места, где остановился.

## Принципы

1. **Уважение к новичку.** На каждом шаге: что сейчас произойдёт и зачем, сколько времени/места,
   выбор `[Продолжить] / [Пропустить] / [Подробнее] / [Выход]`. Enter = рекомендованный вариант.
2. **Защита от дурака.** Валидация ввода (переспрос при ошибке), необратимое — только с явным
   подтверждением, предпроверки ресурсов ДО скачивания (диск, VRAM, сеть, права).
3. **Предусмотрительность.** Заранее считает: хватит ли места/VRAM, есть ли драйвер/CUDA/ROCm,
   Docker/WSL2, интернет; если нет — понятно объясняет и предлагает решение.
4. **Устойчивость.** Машина состояний + чекпоинты: можно прервать/перезагрузиться/запустить снова —
   мастер видит состояние и продолжает. Скачивания докачиваются (уже есть в deploy.mjs, `.part`).
5. **Мультиязычность.** Весь текст мастера — через словарь i18n (старт: RU/EN), легко добавить язык.
6. **Минимум сущностей (KISS).** Один движок скачивания (deploy.mjs). Мастер — только слой диалога,
   детекта и оркестрации. Zero-deps Node, как весь tooling KLAS.
7. **Идемпотентность.** Повторный/возобновлённый запуск на живой системе безвреден.

## Архитектура

```
tools/install.mjs            ← МАСТЕР (интерактив): язык, детект, вопросы, оркестрация, resume
  ├─ tools/lib/i18n.mjs      ← словари RU/EN (ключ → строка), t(key, vars)
  ├─ tools/lib/prompt.mjs    ← readline-хелперы: select/confirm/text с дефолтами и валидацией
  ├─ tools/lib/detect.mjs    ← детект железа/окружения (GPU, VRAM, драйвер, CUDA/ROCm, Docker/WSL2,
  │                            Node, git, winget, свободное место, сеть)
  ├─ tools/lib/state.mjs     ← чекпоинты: .deploy-state.json (фаза, ответы), read/mark/resume
  └─ вызывает:
       • tools/deploy.mjs (движок манифеста: engine/model/docker/llama-swap; шаги как функции)
       • tools/anonymize.mjs (если выбрана анонимность)
       • tools/install-desktop-shortcuts.ps1 (идея 13 — ярлыки в конце)
       • tools/health-check.ps1 (финальная проверка) + тестовый запрос к модели
```

**Состояние** — `.deploy-state.json` (gitignore): `{ lang, phase, answers:{anonymous,libs,gpu,model,zim}, done:[...] }`.
При старте: файл есть → «Продолжаем установку (шаг N из M)…» и идём с незавершённой фазы.

**Возобновление после перезагрузки.** Шаги, требующие reboot (драйвер GPU, CUDA/ROCm-рантайм,
включение WSL2/Docker): (а) фиксируем чекпоинт; (б) объясняем, зачем перезагрузка; (в) ставим
одноразовый resume — `HKCU\...\RunOnce` (или ярлык в Startup) на повторный запуск мастера, ЛИБО
пользователь просто снова запускает ту же команду. После входа мастер читает состояние и продолжает.
⚠️ Persistence (RunOnce/Startup) — с явного согласия пользователя (classifier запрещает агенту
создавать автозапуск молча; здесь это делает САМ установщик по кнопке пользователя).

## Поток мастера (сценарий)

1. **Приветствие + выбор языка** (RU/EN). Короткое «что такое KLAS и что сейчас будет».
2. **Детект окружения** (read-only, быстро): GPU и VRAM, драйвер, CUDA/ROCm, Docker/WSL2, Node, git,
   winget, свободное место, интернет. Показать таблицу «что нашли». 
3. **Диагноз GPU-пути:** NVIDIA → CUDA-сборка движка; AMD → ROCm; нет GPU → предупредить (CPU-режим
   медленный) и спросить, продолжать ли. Если драйвер/рантайм отсутствует — предложить установить
   (winget/оф. установщик), пометить как «шаг с перезагрузкой».
4. **Вопросы установки** (с дефолтами):
   - Анонимный деплой? (да → `anonymize.mjs`; нет → идентификаторы владельца).
   - Качать библиотеки/опциональные компоненты? (mcp, доп. модели).
   - Что из тяжёлого качать: движок (обяз.), модель (выбор из манифеста), .zim база знаний (набор/пропустить).
   - Куда ставить (по умолчанию текущий корень).
5. **Смета и подтверждение:** «Скачаю X ГБ, займёт ~Y места, ~Z минут. Продолжить?» — до любого скачивания.
6. **Установка по фазам** (каждая — чекпоинт): предвызовы драйверов (с reboot-resume) → движок →
   модель → docker-стек → llama-swap → пост-настройка.
7. **Пост-настройка:** ярлыки на Рабочем столе (идея 13), предложить автозапуск (ставит пользователь
   кнопкой), финальный health-check + тестовый запрос к модели.
8. **Финал:** экран успеха — ссылки (пульт `http://localhost/`, чат, wiki), что дальше, где пароль.

## Открытые вопросы (уточнить по ходу, не блокеры)

- ROCm на Windows: официальная поддержка ограничена — для AMD, возможно, CPU/Vulkan-путь как запасной;
  уточнить при Ф2.
- Кросс-платформа (Linux) — горизонт; машина состояний проектируется платформо-нейтрально, детект/шаги
  инкапсулированы в `detect.mjs`/обработчиках.
- Модель по умолчанию — итог Фазы 2b исследований (сейчас в манифесте gemma-4-12b; основная qwen3.5-35b).
