## Стержень

Jarvis/Joi существует, чтобы **помогать и оберегать владельца**. Идентичность/доверие — не «замок на
входе», а **непрерывная адаптивная оценка доверия** (continuous adaptive risk-based auth), которая
поднимает планку проверки при росте риска и **всегда ставит жизнь владельца выше данных**.

## Принципы (PHILOSOPHY.md + заметки владельца + интервью 003)

- **Не изобретать:** собрать из зрелых кирпичей — голосовая биометрия (ECAPA/SpeechBrain), стилометрия,
  adaptive risk auth, duress-коды. (research 08).
- **Голос ≠ доказательство:** он подделывается (реплей/дипфейк — KLAS сам клонирует голоса!). Всегда
  многофакторность + anti-spoofing + step-up для чувствительных действий.
- **Вероятностность, не бинарность:** score+порог, деградация в проверочные вопросы, а не молчаливый замок.
- **Парето 80/20:** быстрые, простые, точные механизмы; глубину наращивать по мере надобности.
- **Приватность (стержень KLAS):** отпечатки/профили/секреты — только локально, зашифрованно; изоляция
  от памяти агентов VS Code (idea 05).
- **Стек — по критерию интервью 003 (⚠️):** голос/стилометрия — вероятно Python (SpeechBrain/pyannote);
  политика доверия/профили — там, где естественнее (Node/control-API Jarvis Gateway).

## Развилки для будущего `/interview` (владельческое, чувствительное — НЕ решаю сам)

1. Матрица «чувствительность действия × требуемый порог доверия» (что можно на каком уровне).
2. Формат duress-сигнала(ов) и реакции (декой-режим / скрытый алерт / отказ / контр-отмена).
3. «Круг помощи владельцу»: кого из близких включаем и какие факты — аутентификаторы.
4. Границы автономии Jarvis при обнаружении принуждения (что он вправе сделать сам).

## Фаза 1 — Профили и модель доверия (каркас, без биометрии) 🔲
**Цель:** структура профиля пользователя + движок trust-score + матрица политики. Работает ещё до голоса.

- [ ] **Профиль пользователя** (локальный, зашифрованный): id, имя, baseline-признаки (заполняются позже),
      библиотека секретов/фактов (хеши/шифр), уровень доверия, заметки, связи с другими профилями.
- [ ] **Владелец = дефолтный максимум доверия**; профиль владельца — первичный.
- [ ] **Движок trust-score:** непрерывная величина на сессию; факторы (пока: устройство/узел tailnet,
      контекст, знание секретов) складываются в score.
- [ ] **Матрица политики «доверие × чувствительность действия»:** каждое действие Jarvis имеет требуемый
      порог; не хватает → step-up или отказ. Начать с 3–4 уровней действий (безобидное → конфиденциальное →
      опасное/необратимое).
- [ ] **Перекрёстные связи** профилей (отношение к юзеру зависит от связей) — простая графовая модель.

**Проверка:** сценарии на бумаге/тестах: владелец (высокое доверие) → всё; неизвестный (низкое) → только
безобидное; запрос секрета при низком доверии → отказ/step-up.

## Фаза 2 — Голосовая биометрия + anti-spoofing (близкая дистанция) 🔲
> Зависит от аудио-тракта Jarvis (research 07: STT/микрофон).

- [ ] **Enrollment владельца:** собрать эталонные speaker-эмбеддинги (ECAPA-TDNN, SpeechBrain).
- [ ] **Верификация на каждом обращении:** эмбеддинг → косинус с эталоном → вклад в trust-score.
- [ ] **Anti-spoofing (liveness/replay/deepfake):** детектор (ASVspoof-класс: AASIST/RawNet2) — голос,
      не прошедший liveness, НЕ повышает доверие; для чувствительных действий — обязательный step-up.
- [ ] **Калибровка порогов** под реальный тракт (шум, Bluetooth-SCO) — измерить FAR/FRR на своих данных.
- [ ] **1:N идентификация** (кто из known-профилей говорит) + диаризация (pyannote), если в комнате несколько.

**Проверка:** владелец опознаётся по голосу (score высокий); запись/дипфейк голоса ловится anti-spoofing
и НЕ даёт доступа к чувствительному без step-up.

## Фаза 3 — Стилометрия речи (средняя дистанция) 🔲
- [ ] **Стилометрический профиль** из транскриптов (STT): частоты функциональных слов (старт, Парето),
      обороты, «особые слова», длина/структура фраз.
- [ ] **Authorship verification:** сравнение текущей манеры с baseline → вклад в trust на средней дистанции.
- [ ] Начать с простого (частоты + порог), усложнять при нужде.

**Проверка:** система замечает несоответствие манеры речи (даже при похожем голосе) → повышает риск.

## Фаза 4 — Красная линия: KBA и step-up вопросы 🔲
- [ ] **Библиотека секретов/фактов** владельца (и, ограниченно, близких) — зашифрованно.
- [ ] **Верификационные вопросы:** при падении доверия Jarvis задаёт аутентификационные вопросы; успех →
      поднимает доверие, провал → резко снижает.
- [ ] **Аутентификация близких «в помощь владельцу»:** ограниченный доступ для валидированного
      друга/родственника к действиям, нацеленным на помощь владельцу (не полный доступ).

**Проверка:** при подозрении Jarvis корректно задаёт вопросы; близкий по фактам биографии получает
ограниченный «помощь-владельцу» доступ; чужой — нет.

## Фаза 5 — Контур принуждения (duress) 🔲 — жизнь важнее данных
> Самый чувствительный контур. Границы поведения — через `/interview` (развилка 2,4).

- [ ] **Duress/panic-сигнал(ы):** кодовое слово/фраза/особый ответ на верификацию → тихий флаг «под угрозой».
- [ ] **Детекция принуждения без кода:** стресс/страх в голосе + нетипичные формулировки + «невозможные»/
      опасные запросы + давление → повышают риск даже при совпадении голоса.
- [ ] **Реакции (жизнь > данные):** внешне имитировать послушание; не выполнять реально опасные/необратимые
      команды; **декой-режим** (безобидное вместо секретов); **тихий отложенный алерт** доверенным; лог.
- [ ] **Контр-отмена** (слово владельца для отмены ложного алерта) + калибровка порогов (ложные срабатывания
      дороги).

**Проверка (на безопасных сценариях-симуляциях):** duress-сигнал переводит Jarvis в защитный режим (декой +
скрытый алерт, отказ от опасного) при внешней видимости послушания; контр-отмена работает.

## Фаза 6 — Мыслительные паттерны + зрелость (далёкий горизонт) 🔲
- [ ] **Семантическая модель мировоззрения** пользователя в памяти Jarvis (убеждения/ценности/темы) —
      накопительная долгодистанционная сигнатура; проверка согласованности новых высказываний.
- [ ] Слияние всех сигналов (голос + стилометрия + знание + устройство + контекст + мышление) в единый
      непрерывный trust-score с объяснимостью («почему снизил доверие»).

## Зависимости и риски
- **Зависимости:** базовый Jarvis (STT/аудио — research 07; память/навыки — idea 05). Фаза 1 (каркас
  доверия) реализуема раньше, без голоса.
- **Риск ложных срабатываний:** биометрия/стилометрия вероятностны (FAR/FRR). Митигация: многофакторность,
  деградация в вопросы, не молчаливый блок (кроме опасного), объяснимость.
- **Риск голосового дипфейка:** KLAS сам клонирует голоса → атакующему это доступно. Митигация:
  anti-spoofing + многофакторность + step-up обязательны.
- **Риск duress-контура:** ложный алерт дорог, пропуск угрозы опаснее. Митигация: калибровка, контр-отмена,
  границы автономии через `/interview`.
- **Приватность:** всё локально/зашифрованно; изоляция от контекста агентов VS Code.
