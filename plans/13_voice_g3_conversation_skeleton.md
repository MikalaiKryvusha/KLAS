# План 13 — Голосовой тракт Г3: «Скелет разговора» (микрофон → ядро → динамики)

> Фаза Г3 голосового блока. Зонтик — [plans/07](07_jarvis_gateway_and_client.md); предыдущие —
> [Г1 «Голос»](11_voice_g1_tts_silero.md), [Г2 «Уши»](12_voice_g2_stt_gigaam.md). Мозг —
> принятое ядро OpenClaw ([plans/10](10_openclaw_core_rollout.md), гейтвей в tailnet).
> Статус: 🟡 **каскад собран и проверен автономно (2026-07-17)** — тёплый ход 20–22 с (критерий
> <25 с взят), контекст сессии держится (тест «запомни 42»). Ждёт 🏠 живой микрофонной проверки
> владельцем: `node tools/voice-talk.mjs` (push-to-talk, Enter старт/стоп; выход — пустая реплика).

## Цель

**Первый живой голосовой разговор с KLAS на самом ПК:** push-to-talk скрипт слушает микрофон,
распознаёт (Г2), отправляет текст в ядро OpenClaw, озвучивает ответ (Г1). Сквозная латентность
замерена. Без телефона, без wake word, без barge-in — скелет.

## Принцип (KISS)

БЕЗ Pipecat на этом шаге: прямой каскад в одном скрипте (`STT → openclaw agent → TTS`) — минимум
сущностей, максимум прозрачности. Pipecat (barge-in, turn detection, WebRTC) подключаем в Г4/Г5,
когда пойдёт realtime-стриминг на устройства — так каркас докажет необходимость (PHILOSOPHY).

## Шаги

- [x] **Каскад** `tools/voice-talk.mjs`: push-to-talk (Enter старт/стоп, запись ffmpeg dshow
      16 кГц, мик по умолчанию «Микрофон (NVIDIA Broadcast)», `--device` для смены) → Г2-STT →
      ход ядра (`openclaw agent --session-key voice-<дата>`, контекст в сессии) → Г1-TTS →
      динамики. Автономный режим `--wav <файл>` — прогон каскада без микрофона (харнесс).
- [x] **Латентность по этапам (замер 2026-07-17):** холодный ход 64 с (свап модели + первый ход
      сессии); **тёплый ход 20–22 с** (уши 1.5 · ядро ~7 · рот 4.2) — критерий <25 с ВЗЯТ.
- [x] **Голосовой стиль:** префикс-инструкция в каждом ходе (константа `VOICE_STYLE`) — ответы
      короткие/разговорные, подтверждено тестом (правка IDENTITY.md не понадобилась — KISS).
- [x] **Контекст сессии:** «запомни: любимое число 42» → следующий ход «какое моё число?» → «42» ✓.
- [x] **Строка в харнесс-таблицу** `AGENT_GUIDE.md`.
- [ ] 🏠 **Живая микрофонная проверка владельцем:** `node tools/voice-talk.mjs` — сказать пару
      фраз push-to-talk; вписать сюда впечатление/дефекты (мик по умолчанию NVIDIA Broadcast).

## Проверка (критерий готовности)

✅ Автономный каскад: синтезированный вопрос → распознан → ядро ответило по-русски кратко →
ответ озвучен; 3 связных хода одной сессии; тайминги напечатаны. Остался живой микрофон (🏠).

## Риски

- **Латентность ядра** (~15 с тёплый ход) — для «скелета» приемлемо, для живого диалога нет.
  Пути ускорения (Г5+/отдельное решение): меньшая модель для голосовых интентов, стриминг ответа
  в TTS по предложениям, кеш системного промпта. НЕ оптимизировать преждевременно.
- Запись микрофона в Node на Windows — если пакеты записи хрупкие, сайдкар: sherpa-onnx mic-пример
  или ffmpeg (dshow) как записыватель.
