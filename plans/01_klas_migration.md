# План 01 — Миграция LOCAL_NOMAD → F:\KLAS (рождение KLAS)

> Реализует `ideas/01_origin_KLAS.md` (Фаза 0.5 мастер-плана). Статус: 🔧 скрипт готов, ждёт запуска
> ВЛАДЕЛЬЦЕМ. После проверенной миграции — DONE-тег этому плану и идее 01.

## Цель

Всё нужное из `F:\LOCAL_NOMAD\` умно переезжает в `F:\KLAS\`; проект (документы KAIF + git-история)
становится корнем `F:\KLAS\`; пути и имена заменяются; «личность» Claude Code (память проекта)
мигрирует; устаревшее остаётся в `F:\LOCAL_NOMAD\`. После миграции агент в новой сессии ничего не
теряет.

## Целевая раскладка F:\KLAS\

Корень = git-репозиторий (нынешнее содержимое `local_ai_agent\`), инфраструктура — поддиректории с
**прежними именами** (KISS: вся замена путей = `F:\LOCAL_NOMAD → F:\KLAS`, скрипты и конфиги не
ломаются).

## Что делает скрипт `tools/migrate-to-klas.mjs`

По умолчанию — **DRY-RUN** (только печатает план, ничего не меняет). Реальное выполнение — `--apply`.

1. **Предпроверки:** источник существует (маркер `.kaif`); в `F:\KLAS\` нет чужого `.git` (мы приносим
   свою историю) и нет коллизий имён с переезжающим — **собственные файлы владельца в `F:\KLAS\`
   (логотип `8O-T3TotiIs.jpg`, `KLAS_LOGO_GOAL.txt` → см. `ideas/02_klas_logo_og.md`) сохраняются как
   есть**; предупреждения, если запущены `llama-server.exe` или docker-контейнеры (файлы могут быть
   заблокированы); предупреждение о незакоммиченных изменениях.
2. **Перемещение** по манифесту (`rename` на одном диске = мгновенно; fallback copy+delete).
3. **Замена путей** во всех текстовых файлах `F:\KLAS\` (`.md .bat .cmd .ps1 .yml .yaml .json .mjs
   .js .txt`, `Caddyfile`; бинарные директории `.git`, `LLMs`, `kiwixdb`, exe/dll llamacpp
   пропускаются): `F:\LOCAL_NOMAD\local_ai_agent → F:\KLAS` и `F:\LOCAL_NOMAD → F:\KLAS`
   (оба разделителя `\` и `/`, любой регистр буквы диска).
4. **Комментирует блок `anythingllm`** в `docker-compose.yml` (сервис остался в LOCAL_NOMAD).
5. **Миграция «личности» Claude Code:**
   - копирует `C:\Users\krinik\.claude\projects\f--LOCAL-NOMAD-local-ai-agent\` в `...\f--KLAS\` и
     `...\F--KLAS\` (оба варианта регистра диска — Claude Code строит имя папки из cwd), с заменой
     путей в `*.md` (память); оригинал остаётся как бэкап;
   - в `~/.claude.json` (с бэкапом `~/.claude.json.backup-klas`) клонирует запись проекта
     `F:\LOCAL_NOMAD\local_ai_agent` под ключи `F:\KLAS` и `f:\KLAS`.
6. **Git remote:** в `F:\KLAS\` ставит `origin = https://github.com/MikalaiKryvusha/KLAS.git`.
7. **Отчёты:** пишет `F:\KLAS\MIGRATION_REPORT.md` (что перемещено/оставлено/заменено + следующие
   шаги) и `F:\LOCAL_NOMAD\README_MOVED.md` (указатель «система переехала в F:\KLAS»).

## Инструкция для владельца

```powershell
cd F:\LOCAL_NOMAD\local_ai_agent

# 1) Репетиция (ничего не меняет, печатает полный план):
node tools\migrate-to-klas.mjs

# 2) Если план устраивает — закрой VS Code с этим проектом, останови llama-server и docker,
#    затем из ЛЮБОГО другого терминала:
node F:\LOCAL_NOMAD\local_ai_agent\tools\migrate-to-klas.mjs --apply

# 3) Открой VS Code в F:\KLAS\ и скажи агенту: «продолжи» (/resume)
```

## Проверка после миграции (первая сессия в F:\KLAS — чек-лист агенту)

1. `npm run kaif:check` — структура KAIF целостна.
2. `git status` + `git log --oneline -5` — история на месте; `git remote -v` — origin = KLAS.
3. Прочитать `MIGRATION_REPORT.md`; выборочно проверить замену путей (`gemma4-12b.bat`,
   `docker-compose.yml`, карты).
4. Память Claude Code жива (агент помнит контекст работы).
5. Первый push: `git push -u origin main`.
6. DONE-теги: `git mv ideas/01_origin_KLAS.md ideas/01_DONE_origin_KLAS.md`, аналогично этому плану;
   статус-секции внутрь; обновить `STATUS.md`.
7. `F:\LOCAL_NOMAD\` не удалять без явного решения владельца (там остались AnythingLLM,
   OLLAMA_MODELS, `1.txt`).

## Риски (Мёрфи)

- **(а) Заблокированные файлы** (llama-server/docker запущены) → предпроверка предупреждает; запускать
  при остановленных сервисах.
- **(а) Регистр буквы диска в имени папки памяти Claude Code** → создаём оба варианта (`f--KLAS` и
  `F--KLAS`), лишний безвреден.
- **(б) VS Code держит открытым старый путь** → инструкция: закрыть VS Code перед `--apply`.
- **(б) docker-compose: пути томов** → блок anythingllm комментируется; остальные тома переезжают
  вместе с данными (kiwixdb, homepage, caddy).
- **(в) Сессионные логи чатов (`*.jsonl`) содержат старые пути внутри** → это история, не трогаем;
  на работу не влияет.
