# План 11 — Голосовой тракт Г1: «Голос» — русский TTS (Silero v5)

> Фаза Г1 голосового блока эпика ассистента. Зонтик — [plans/07](07_jarvis_gateway_and_client.md)
> (нарезка Г1–Г5), эпик — [plans/08](08_assistant_epic_roadmap.md). Решения: интервью 004 Q3
> (Silero v5 — стартовый голос до клонов), интервью 003 Q5-принцип (аудио/ML — Python-сайдкар).
> Статус: 🔧 в работе (2026-07-17).

## Цель (одним предложением)

KLAS умеет **говорить по-русски локально**: команда `node tools/voice-say.mjs "текст"` синтезирует
речь Silero v5 (CPU, офлайн) в wav и/или динамики, латентность синтеза замерена и записана.

## Почему Г1 первая (перед «ушами»)

TTS не требует входных данных → его выход становится **тестовыми данными для СТТ (Г2)**: round-trip
«текст → речь → текст» проверяется автономно, без человека и микрофона.

## Шаги

- [ ] **Дом голосового тракта:** `F:\KLAS\voice\` (инфраструктурная директория ВНЕ git, как llamacpp/);
      в git — только скрипты `tools/voice/*` и конфиги. Python venv: `F:\KLAS\voice\venv` (Python 3.14
      системный; если колёса torch под 3.14 не готовы — поставить рядом 3.12/3.13 только для venv).
- [ ] **Silero v5 ru:** пакет/веса (~50–150 МБ, MIT), загрузка в `F:\KLAS\voice\models\`.
- [ ] **Сайдкар-скрипт** `tools/voice/silero_say.py`: stdin/арг текст → wav (+ `--play` в динамики,
      `--voice` выбор из русских голосов, ударения из коробки). Комментарии по канону.
- [ ] **Node-обёртка** `tools/voice-say.mjs` (единый вход в харнесс KLAS): вызывает сайдкар venv,
      печатает тайминги (загрузка модели / синтез / длительность аудио).
- [ ] **Замер:** латентность синтеза короткой фразы (тёплый старт), RTF; строка в таблицу харнесса
      `AGENT_GUIDE.md`.
- [ ] **Выбор дефолтного голоса** (мужской для Jarvis-строя из 29 доступных) — слушает владелец
      (🏠 мини-домашка), временный дефолт ставлю сам.

## Проверка (критерий готовности)

`node tools/voice-say.mjs "Привет, Криник! Я — КЛАС, твой локальный ассистент."` → корректный
русский wav (без роботизации, с ударениями), синтез быстрее реального времени на CPU; файл
проигрывается; тайминги в выводе.

## Риски

- **Python 3.14 vs torch:** если PyTorch-колёс под 3.14 нет — отдельный Python 3.12/3.13 для venv
  (только внутри `voice\`, систему не трогаем). Альтернатива — ONNX-вариант Silero.
- Silero v5 API мог смениться против v4 — сверяться с оф. репо (живость: silero-models активен).
