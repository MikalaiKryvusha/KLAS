# `plans/` — детальные планы шагов

Детальные планы отдельных кусков работы: шагов мастер-плана, конкретных фич, идей, багов,
исследований, процедур. **`MASTER_PLAN.md`** (корень проекта) — высокоуровневая дорожная карта;
`plans/` держит приближённые планы, реализующие её шаги. Один `NN_<name>.md` на план.

**Человеку (владельцу):** писать сюда не обязательно — планы обычно пишет агент. Можешь положить план,
если хочешь управлять тем, *как* что-то делается. Читай их, чтобы видеть намеченный подход агента до
того, как он его исполнит.

**ИИ-агенту:** перед нетривиальной работой напиши сюда короткий план и следуй ему. Нумеруй файлы
(`NN_<name>.md`). Завершённый и проверенный план получает тег `DONE` в имени файла
(`git mv NN_x.md NN_DONE_x.md`) плюс статус-секцию. Справочные материалы (не закрываемые задачи)
DONE не помечаются.
