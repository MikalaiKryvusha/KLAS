## Ключевые выводы для мастер-плана

1. **Стек по факту один** (llama.cpp + gemma-4-12b + Zoo Code) — остальное окружение (docker-сервисы)
   ортогонально и не мешает, кроме конфликта порта 8080 (проверить!).
2. **Два сильных подозреваемых по багу владельца** (ошибки переполнения контекста): завышенный
   `-c 256000` и конфликт порта с kiwix. Детали и план проверки — `bugs/01_context_overflow_zoo_code.md`.
3. **VRAM-бюджет** — центральное ограничение: 16 GB = модель (7.4 GB у gemma) + KV-кеш (растёт с `-c`)
   + оверхед. Расчёт корректного `-c` — задача Фазы 1.
4. Для «пульта» (Фаза 5) уже есть заготовка — homepage в docker; для «спит, пока не позовут»
   (Фаза 4) — nssm. Не плодить новые сущности, пока не проверены имеющиеся (Оккам).
5. Qwen3.6-27B в текущем виде — неудачный кандидат для 16 GB VRAM; вопрос лучшей модели — в
   исследование Фазы 2 (`researches/02_optimal_stack_2026.md`, будет создан).
