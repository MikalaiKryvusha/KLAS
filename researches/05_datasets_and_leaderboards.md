## Часть 1. Что такое датасеты (и `Glint-Research/Fable-5-traces`)

**Датасет** — набор данных для обучения/дообучения/оценки моделей. Для LLM это обычно пары
«вход → желаемый выход» (или диалоги/трейсы), на которых модель учат поведению.

**`Glint-Research/Fable-5-traces`** — официальный датасет **трейсов агента Fable 5** (на базе Claude):
- **4 665 сессий** реального агентного кодинга: промпты, цепочки рассуждений (`cot`), вызовы
  инструментов (Bash, Read, Edit, Write, web, task-management) и результаты. 81% строк — вызовы
  инструментов; медиана рассуждения ~2365 символов. Формат: `pi-traces/*.jsonl` + плоский
  `fable5_cot_merged.jsonl` (188 МБ). Лицензия **AGPL-3.0**.
- **Назначение:** SFT (supervised fine-tuning), **дистилляция** рассуждений и tool-use-политики в
  меньшие модели, обучение агентскому поведению.

**Чем полезен KLAS:** это готовый учебный материал, чтобы **дообучить локальную модель (9–35B) вести
себя как агент** — правильно звать инструменты, рассуждать перед действием. Потенциальная будущая
фаза KLAS: взять нашу основную модель и дообучить/дистиллировать на таких трейсах под агентский
кодинг в Zoo Code. ⚠️ AGPL-3.0 обязывает открывать исходники при распространении — учитывать, если
будем публиковать дообученную модель. Файнтюн — тяжёлая GPU-задача (пока за горизонтом MVP, но
направление зафиксировано).

## Часть 2. Лидерборды HF — программный доступ (инструмент `tools/leaderboard.mjs`)

HF ведёт **официальные бенчмарки** с программным API — не нужно парсить веб-страницы:

| Запрос | Что даёт |
|--------|----------|
| `GET /api/datasets?filter=benchmark:official` | список официальных бенчмарков (SWE-bench Verified, HLE, MMLU-Pro, terminal-bench, …) |
| `GET /api/datasets/{id}/leaderboard` | ранжированный список моделей с баллами (rank, modelId, value, verified) |
| `GET /api/models/{id}` | метаданные модели: размер (`safetensors.total`), дата релиза (`createdAt`) |
| датасет `OpenEvals/leaderboard-data` (parquet) | кросс-бенчмарк баллы одним файлом |

**Актуальность:** API живой на июль 2026, свежие модели попадают (в SWE-bench Verified топ —
`Ornith-1.0-397B` 82.4; из наших размеров Qwen3.6-27B #9 = 77.2). Это заменяет устаревший
community-рейтинг из `researches/02`.

**Наш инструмент — `tools/leaderboard.mjs`** (реализует «всегда знать лучшие свежие модели ≤16 ГБ»):
```
node tools/leaderboard.mjs                      # SWE-bench Verified, модели ≤ 35B, с датами
node tools/leaderboard.mjs --bench cais/hle --max-params 32
node tools/leaderboard.mjs --list               # какие бенчмарки есть
```
Тянет лидерборд, обогащает размером+датой, фильтрует под наше железо. Замер 2026-07-03 (≤35B,
SWE-bench Verified): Qwen3.6-27B (77.2, апр 2026), **Ornith-1.0-35B (75.6, июнь 2026 — новинка,
кандидат на `try-model`)**, IndustrialCoder-32B (74.8), Qwen3.5-27B (72.4).

## Рабочий цикл поиска моделей (идея 06 + 07)

1. `node tools/leaderboard.mjs` — увидеть свежий топ под наше железо (обновляется само из HF).
2. Выбрать кандидата → найти GGUF-квант (unsloth/bartowski и т.п.) под 16 ГБ (см. `researches/03`).
3. `node tools/try-model.mjs <repo> <файл.gguf>` — скачать + бенч скорости.
4. Добавить в `llama-swap/config.yaml` → `node tools/agent-bench.mjs <alias>` — бенч ума.
5. Результат — в таблицу `researches/02`. Периодичность — раз в 1–2 недели (пул STATUS).
