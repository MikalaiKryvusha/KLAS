# Bug 01 — Zoo Code падает с ошибкой переполнения контекста (llama.cpp + gemma-4-12b)

**Статус:** 🟡 ФИКС ПРИМЕНЁН (2026-07-03) — ждёт подтверждения владельцем в реальной сессии Zoo Code
**Версия/сборка:** llama.cpp (CUDA 13, версия не зафиксирована) + `gemma-4-12b-it-UD-Q4_K_XL.gguf` + Zoo Code (профиль `gemma4-12b-llamacpp`)
**Когда/контекст:** сообщено владельцем в `GOAL.md`; заведено 2026-07-02 при деплое KAIF

## Симптом

Иногда Zoo Code сообщает об ошибке llama.cpp: модель не смогла обработать запрос, так как запрос
превысил контекстное окно модели. Ожидаемое поведение: запросы обрабатываются стабильно, без ошибок
переполнения контекста (приоритет №1 владельца — стабильность).

## Воспроизведение (детерминированное)

Пока НЕ детерминировано («иногда»). План воспроизведения:
1. Запустить сервер: `F:\KLAS\llamacpp\bat\gemma4-12b.bat`.
2. Снять факты: `curl http://127.0.0.1:8080/health`, `curl http://127.0.0.1:8080/props`
   (какой контекст РЕАЛЬНО выделен), `nvidia-smi` (VRAM), `netstat -ano | findstr :8080` (кто на порту).
3. Отправить в `/v1/chat/completions` запрос с очень длинным промптом (скриптом, нарастающая длина) —
   найти порог, на котором возникает ошибка, и её точный текст.

## Криминалистика

- Точный текст ошибки Zoo Code не зафиксирован — попросить владельца скопировать при следующем
  возникновении, либо снять с харнесса при воспроизведении. Факты конфигурации — в
  `researches/01_local_nomad_audit.md`.
- **2026-07-02 (наблюдение):** docker-контейнеры ЗАПУЩЕНЫ — `anythingllm`,
  `caddy`, `kiwix_wikipedia`, `homepage`. Значит kiwix реально держит порт 8080 → **гипотеза B
  (конфликт порта с llama-server) подтверждена наполовину**: осталось проверить, что происходит при
  одновременном запуске `gemma4-12b.bat` (`netstat -ano | findstr :8080`).
- **2026-07-03 (из researches/02, llama.cpp issue #17284):** llama-server при переполнении контекста
  НЕ падает — возвращает **HTTP 400** «request exceeds the available context size». Это усиливает
  гипотезу A: `-c 256000` не влезает в VRAM (KV-бюджет), а «падение» Zoo Code — необработанный
  400-ответ. Дополнительно: KV-кеш ≈ +25% к весам на 8K ctx и +100% на 32K; `--jinja` обязателен
  для tool calling; KV-квант `q8_0` (не `q4_0` — портит агентику).

- **2026-07-03 (диагностическая сессия агента, харнесс `tools/health-check.ps1`):**
  - Сборка llama.cpp — **b9538**: Blackwell нативно (ARCHS 1200/1210, FP4), есть авто-fit памяти.
  - **`n_ctx_train = 262144`**: у gemma-4-12b нативный контекст 256K → `-c 256000` в пределах
    возможностей модели. Гипотеза A в исходной формулировке **опровергнута** для этой сборки:
    сервер поднялся, `/props` показал честные `n_ctx=256000`, генерация работала.
  - Цена 256K: VRAM 14.7/16.3 GB (запас всего ~1.6 GB) — на грани; риск нестабильности под
    нагрузкой. При `-c 131072`: VRAM 12.5/16.3 GB (запас ~3.8 GB), KV целиком на GPU.
  - Замеры (256K-конфиг): короткий запрос — 68 t/s генерация; промпт 25 264 токенов — 7.8 с
    (3434 t/s prompt), 63 t/s генерация, память стабильна. Русский язык чистый.
  - Порт: на свежей загрузке docker контейнеры сам НЕ поднимает; но когда поднимет — kiwix
    забирает host-8080. **Гипотеза B остаётся главным объяснением симптома**: Zoo Code стучался
    в kiwix вместо llama-server (или сервер не мог забиндить порт).

## Корневая причина / Гипотезы (ранжированы)

- **A (главная): завышенный `-c 256000` в `gemma4-12b.bat`.** Если 256000 превышает врождённый лимит
  модели или KV-кеш не помещается в 16 GB VRAM — llama-server может урезать/не выделить контекст, и
  реальный лимит окажется меньше заявленного, а Zoo Code, считая контекст большим, шлёт запросы «с
  запасом» → ошибка переполнения. Проверка: `/props` покажет фактический `n_ctx`.
- **B: конфликт порта 8080.** Docker-kiwix маппит 8080; если llama-server (дефолтный порт 8080) и
  kiwix запущены одновременно — один из них не встаёт или Zoo Code стучится не туда. Проверка:
  `netstat -ano | findstr :8080` при запущенном Docker и сервере.
- **C: несогласованность настроек Zoo Code.** Заявленный в профиле Zoo Code размер контекста больше
  фактического у сервера; или Zoo Code не учитывает токены ответа/системного промпта. Проверка:
  сравнить настройки профиля с `/props` (возможно, потребуется домашка владельцу).
- **D (вклад в «глупость», не в падения): `--reasoning off` и агрессивные penalty
  (repeat-penalty 1.2 + frequency 0.15 + presence 0.2)** — глушат рассуждения и могут портить
  качество кода. Отметить для Фазы 1/2.

Правило инварианта (внутренняя карта): контекст фронтенда ≤ контекст движка ≤ лимит модели, и
модель + KV ≤ 16 GB VRAM. Похоже, нарушен хотя бы один уровень этой цепочки.

## План фикса

1. Диагностика по плану воспроизведения (факты, не догадки).
2. Вычислить корректный `-c`: min(лимит модели, что помещается в VRAM с моделью 7.4 GB), с запасом.
3. Править `gemma4-12b.bat`: обоснованный `-c` с комментарием, явный `--port` (не дефолтный 8080,
   чтобы развести с kiwix), зафиксировать изменения здесь.
4. Согласовать профиль Zoo Code с новым фактическим контекстом (если настройки в UI — домашка владельцу).
5. Стресс-тест длинным контекстом через харнесс → критерий приёмки: ни одной ошибки переполнения.

## ✅ Применённый фикс (2026-07-03)

1. **`llamacpp/bat/gemma4-12b.bat` переписан** (все параметры прокомментированы):
   `--port 8080` явно; `-a gemma-4-12b`; **`-c 131072`** (KV целиком в VRAM, запас ~3.8 GB —
   выбор в пользу стабильности; нативный максимум 262144 задокументирован в комментарии).
2. **`docker-compose.yml`: kiwix host-порт 8080 → 8081.** Внутри docker-сети kiwix остался на
   8080, Caddy проксирует на `kiwix_wikipedia:8080` — для веб-доступа через `/wiki` ничего не
   меняется. Конфликт портов устранён навсегда.
3. Харнесс пополнен: `tools/health-check.ps1` (порт, процессы, /health, /props, VRAM — одной командой).
4. **Попутно найдены и починены два дефекта запуска bat:**
   - на машине включена защита Windows `NoDefaultCurrentDirectoryInExePath=1` — cmd НЕ ищет exe в
     текущей папке, поэтому вызов `llama-server.exe` без пути падал «не является командой»; в bat
     теперь ПОЛНЫЙ путь к exe (правило внесено в AGENT_GUIDE);
   - bat-файлы должны быть с CRLF-окончаниями строк — LF ломает парсер cmd (крошит caret-переносы).

**Осталось для закрытия (DONE):** владелец подтверждает отсутствие ошибок в реальной сессии
Zoo Code (профиль Zoo Code должен считать контекст ≤131072 с запасом на ответ; прямой доступ к
википедии теперь `http://127.0.0.1:8081/wiki/` — если есть закладки). Гипотеза D (penalty/reasoning
глушат качество) — отложена до Фазы 2b (сравнение моделей).
