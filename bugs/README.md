# `bugs/` — дефекты, трудности, поломки

Один документ на дефект: симптом, детерминированное воспроизведение, криминалистика, корневая причина /
гипотезы, история фиксов, статус. Собственный долговечный беклог багов агента — ничего не теряется, и
любой баг может быть подхвачен «с холода» будущей сессией. Один `NN_<name>.md` на баг.

**Человеку (владельцу):** можешь завести баг здесь простыми словами (что не так, как воспроизвести);
агент его структурирует. Просматривай директорию, чтобы видеть известные дефекты и их статусы.

**ИИ-агенту:** встретил дефект в ходе работы/тестирования — заведи его здесь по канону (скилл:
`/report-bug`; метод: `BUG_FIXING_FRAMEWORK.md`) — даже мелкий. Пока открыт — без тега `DONE`. Когда
починен **и проверен** — `git mv NN_x.md NN_DONE_x.md` и добавь секцию `## ✅ STATUS: DONE (дата)`.
После 3 неудачных слепых попыток фикса — стоп и переход к исследованию (`/bug-research`).
