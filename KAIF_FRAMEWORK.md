## Что такое KAIF

KAIF (Krinik AI Framework) — **контекстоустойчивый, дисциплинированно-автономный операционный фреймворк
для тандема человек–ИИ**. Он выносит рабочую память и дисциплину агента наружу, в этот репозиторий, —
небольшой набор markdown-документов, конвенций директорий и повторяемых slash-скиллов — так что любая
свежая сессия агента возобновляется с полным контекстом, работает автономно в ясных границах и
накапливает знание вместо того, чтобы терять его. Это не код; это *процесс, зафиксированный файлами,
которые читает агент*.

## Зачем он здесь — что он даёт этому проекту

- **Нет холодных стартов.** Новая сессия читает `AGENT_GUIDE.md` + `STATUS.md` и сразу продуктивна.
- **Знание выживает.** Баги, решения, исследования и идеи становятся долговечными документами, а не
  потерянным чатом.
- **Ограниченная автономия.** Агент молотит беклог сам и эскалирует только решения уровня владельца.
- **Общий метод.** Человек = визионер (`GOAL.md`), агент = исполнитель; KAIF — интерфейс между ними.

## Как это работает здесь — движущиеся части

| Часть | Роль в этом проекте |
|-------|---------------------|
| `AGENT_GUIDE.md` | Канон, который агент читает перед каждой задачей. |
| `PHILOSOPHY.md` | Как мыслит агент (KISS + Оккам + расширенный набор принципов). |
| `BUG_FIXING_FRAMEWORK.md` | Как агент чинит дефекты. |
| `GOAL.md` / `MASTER_PLAN.md` | Видение владельца и фазовый путь к нему. |
| `STATUS.md` | Живое состояние — обновляется после каждой значимой задачи. |
| `PROJECT_STRUCTURE_EXTERNAL_MAP.md` / `PROJECT_ARCHITECTURE_INTERNAL_MAP.md` | Внешняя и внутренняя карты. |
| `plans/ ideas/ bugs/ researches/ interviews/ homeworks/` | Директории знаний (у каждой свой README). |
| `.claude/skills/` | Повторяемые ритуалы (`/resume`, `/pause`, циклы, …) — 19 скиллов. |
| `.kaif/kaif.json` | Маркер деплоя: версия, сфера, агент, tracking. |
| `package.json` + `tools/kaif.mjs` | npm-handles `kaif:*` (version, check, …). |

## Жизнь с KAIF (жизненный цикл)

`/kaif-version` (проверить обновления) · `/kaif-update` (уважительная миграция с origin) ·
`/kaif-fork` (развивать свой) · `/kaif-switch-origin` · `/kaif-remove` (частичное — с сохранением
артефактов, или полное; всегда уважительно). Подкреплено handles `npm run kaif:*`.
