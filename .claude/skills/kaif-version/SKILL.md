# /kaif-version — версия развёрнутого KAIF и проверка обновлений

KAIF разворачивается (инжектируется) в проект конкретной версией. Этот скилл говорит человеку, какая
версия в проекте и есть ли новее в апстриме.

## Что делать

1. **Прочитай локальный маркер** `.kaif/kaif.json`:
   ```json
   { "framework": "KAIF", "version": "X.Y", "released": "YYYY-MM-DD",
     "origin": "https://github.com/MikalaiKryvusha/KAIF", "tracking": "origin", "sphere": "...", "agent": "..." }
   ```
   Сообщи: текущая версия + дата релиза, режим `tracking` (`origin` или `fork`), сфера и агент.
   (Эквивалентная быстрая команда: `npm run kaif:version`.)

2. **Проверь origin на более новый релиз.** Запроси последний релиз/тег origin-репозитория, например:
   ```bash
   gh release view --repo MikalaiKryvusha/KAIF --json tagName,publishedAt 2>/dev/null \
     || gh api repos/MikalaiKryvusha/KAIF/releases/latest --jq '.tag_name + " " + .published_at'
   ```
   Сравни семантические версии (`MAJOR.MINOR`).

3. **Отчитайся человеку:**
   - Если актуально → так и скажи.
   - Если есть новее → скажи какая, и предложи: *«Вижу более новую версию KAIF (vX.Y, ДАТА). Провести
     уважительное обновление-миграцию для этого проекта?»* → если да, передай в `/kaif-update`.
   - Если `tracking` = `fork` → отметь, что проект следует собственному форку KAIF пользователя, а не
     официальному origin; обновления origin — только к сведению (см. `/kaif-switch-origin` для
     возврата на официальный).

## Заметки
- Если `.kaif/kaif.json` отсутствует — KAIF, возможно, здесь не развёрнут (или маркер потерян) — так и
  скажи и укажи на `KAIF.md` для (пере)деплоя.
- Скилл только для чтения: никогда не меняет проект. Обновления — через `/kaif-update`.
