# LOCAL NOMAD — Local AI Agent — Внутренняя карта архитектуры

> **ВНУТРЕННЯЯ карта: логическая архитектура проекта** — объекты-абстракции, в которых работает сфера
> проекта (инженерия локальной ИИ-инфраструктуры), их суть и взаимодействие. Где
> `PROJECT_STRUCTURE_EXTERNAL_MAP.md` говорит *где что лежит*, эта карта говорит *как система мыслит*.
> **Живой справочник — никогда не DONE.**

## Ключевые абстракции

| Абстракция | Что это по сути | Ответственность |
|------------|-----------------|-----------------|
| **Модель (LLM)** | GGUF-файл весов с квантизацией и врождённым лимитом контекста | «ум» системы; определяет качество, требования к VRAM |
| **Движок** | Сервер инференса (сейчас llama.cpp `llama-server.exe`), поднимающий модель как OpenAI-совместимый HTTP API | скорость, размещение модели в VRAM/RAM, реальный размер контекста, слоты |
| **Профиль запуска** | Скрипт/конфиг запуска движка с конкретной моделью и параметрами (сейчас `gemma4-12b.bat`) | единственный источник правды «как именно поднят сервер»: `-c`, `-ngl`, порт, сэмплинг |
| **Агентский фронтенд** | Клиент, превращающий LLM в агента (сейчас Zoo Code в VS Code) | промптинг, инструменты, отправка запросов в API движка; его настройки должны СОГЛАСОВЫВАТЬСЯ с профилем запуска |
| **Харнесс** | Скрипты и команды проверки (health, `/props`, бенчмарк, VRAM, порты) | объективное наблюдение системы без человека |
| **Жизненный цикл** | Механизм «спит, пока не позовут»: запуск по требованию, останов/выгрузка в простое | ресурсы ПК в простое ≈ 0 (Фаза 4) |
| **Пульт (дашборд)** | Веб-интерфейс управления движками/моделями/профилями | удобство владельца (Фаза 5) |
| **Сервисы окружения** | Docker-стек (kiwix, anythingllm, homepage, caddy), Tailscale, MCP | соседние потребители портов/ресурсов; кандидаты в состав решения |
| **База знаний KAIF** | `bugs/`, `researches/`, `interviews/`, `plans/`, `ideas/`, `homeworks/` | накопление знаний проекта между сессиями |

## Как они взаимодействуют

```
Владелец ──видение──▶ GOAL.md ──▶ MASTER_PLAN.md ──▶ plans/ (работа агента)

Агентский фронтенд (Zoo Code / VS Code)
        │  HTTP (OpenAI API, http://127.0.0.1:<port>)
        ▼
Движок (llama-server) ◀──параметры── Профиль запуска (bat)
        │  загружает                        ▲
        ▼                                   │ правится только через bugs/plans
Модель (GGUF, VRAM-бюджет 16 GB)            │
        ▲                                   │
Харнесс ─наблюдает─▶ движок/модель/GPU ─факты─▶ bugs/ и researches/

Жизненный цикл ─старт/стоп─▶ Движок          Пульт ─управляет─▶ Жизненный цикл + Профили
```

Цепочка запроса: владелец пишет в фронтенде → фронтенд шлёт prompt в API движка → движок гоняет модель
в пределах контекста и VRAM → ответ возвращается. Сбой возможен на каждом стыке — поэтому стыки
(настройки фронтенда ↔ параметры профиля ↔ лимиты модели) обязаны быть согласованы явно.

## Инварианты и правила модели

1. **VRAM-бюджет:** размер модели + KV-кеш (зависит от `-c`) + оверхед ≤ 16 GB. Нарушение = падения
   или деградация. Всякое изменение `-c`/модели пересчитывает бюджет (проверка: `nvidia-smi`).
2. **Согласованность контекста:** заявленный контекст фронтенда ≤ фактический контекст движка (`-c`) ≤
   врождённый лимит модели. Нарушение = ошибки «захлебнулся контекстом» (баг 01).
3. **Один порт — один хозяин:** каждый сервис слушает свой явно заданный порт; карта портов — во
   внешней карте. Дефолтные порты не полагаются на удачу (llama-server:8080 vs kiwix:8080).
4. **Стабильность > ум > скорость** — при любом трейд-оффе (порядок приоритетов владельца).
5. **Минимум сущностей:** новый движок/модель/сервис добавляется только с доказанной необходимостью
   и решением владельца (интервью).
6. **Ничего «на глазок»:** утверждение о качестве/скорости/стабильности стека подкрепляется командой
   харнесса или бенчмарком, зафиксированным в документе.

## Ключевые решения, зашитые в архитектуру

- Проект-«управляющий центр» отделён от управляемого окружения `F:\LOCAL_NOMAD\` — знания и конфиги
  версионируются в git, тяжёлые артефакты (модели, dll) — нет. (См. журнал решений в `MASTER_PLAN.md`.)
- Движок общается с фронтендами по OpenAI-совместимому HTTP API — это позволяет менять фронтенд и
  движок независимо (слабая связанность, Фаза 2 может заменить любой элемент).
