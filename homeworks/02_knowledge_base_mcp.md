# Домашка 02 — Подключить поиск по локальной википедии в Zoo Code (MCP-адаптер)

> Реализует `ideas/08` задачу 3b. Инфраструктура настроена агентом 2026-07-03 (openzim-mcp,
> docker-образ готов, конфиг — `mcp/openzim-mcp.json`). От тебя — включить MCP-сервер в Zoo Code.
> Статус: 🟢 готово к подключению.

## Что это даёт

Локальному агенту (Zoo Code) — **инструмент поиска по неограниченной оффлайн-википедии** (наши
~73 ГБ `.zim`: ru-wikipedia, wikibooks, wikisource, wikiversity EN+RU). Агент сможет искать факты в
локальной базе знаний так же, как через веб-поиск, но оффлайн и мгновенно. Это тот самый слой,
«объясняющий» агенту, как искать в локальной базе (идея 08).

## Что уже сделано (агент)

- Выбран готовый инструмент **openzim-mcp** (`cameronrye/openzim-mcp`) — читает `.zim` напрямую,
  оффлайн; docker-образ вытянут и проверен на нашей `kiwixdb/` (сервер стартует, видит базы).
- Конфиг MCP-сервера — `mcp/openzim-mcp.json` (docker-запуск, монтирует `F:/KLAS/kiwixdb`).
- Подготовка одной командой: `node tools/deploy-knowledge.mjs --mcp`.

## Что сделать тебе (в Zoo Code)

1. Открой MCP-настройки Zoo Code (он форк Cline/Roo — секция **MCP Servers**).
2. Добавь сервер `openzim` из `mcp/openzim-mcp.json`:
   ```json
   "openzim": {
     "command": "docker",
     "args": ["run","-i","--rm","-v","F:/KLAS/kiwixdb:/data","ghcr.io/cameronrye/openzim-mcp:latest","--mode","advanced","/data"]
   }
   ```
3. Перезагрузи MCP-серверы в Zoo Code. У агента появятся инструменты: `zim_search` (полнотекстовый
   поиск), `zim_get` (прочитать статью), `zim_query` (умный запрос), `zim_browse`, `zim_metadata` и др.
4. Проверка: попроси агента «найди в локальной википедии статью про …» — он вызовет `zim_search`.

## Заметки

- Поиск идёт по файлам `.zim` в `kiwixdb/` напрямую (docker-kiwix для этого не обязателен — но он же
  отдаёт их людям на `/wiki`).
- Добавить новые базы знаний: `node tools/deploy-knowledge.mjs --list` / `--get <ключ>`.
- Тяжёлые `.zim` в репозиторий KLAS НЕ входят (идея 03) — качаются отдельно, по желанию.
